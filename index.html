FlightPathCore
Cloud-first API testing platform · v1.0.0
FlightPathCore is a unified API testing and orchestration platform. It connects your API collections, swagger schemas, infrastructure, flow diagrams and mapping documents in one place — all derived from your monorepo.
What's in the tool
1
Request Builder — Build and send API requests. Headers pull from Azure Key Vault and App Config automatically.
2
Flow Editor — Drag services onto a canvas, connect them with arrows, and add field mappings between services.
3
Model Mapper — Visual field mapping between two swagger models. Saved per arrow in the Flow Editor.
4
Email Client — View and assert on emails triggered by your C# services running in AKS.
5
fpgit.com — Monorepo browser. Merge a PR and watch Collections, SwaggerHub and Flow update live.
6
C# Orchestration — Write C# test flows with IntelliSense. Actions auto-registered from your collections. Hit ▶ Run Flow to execute with assertions.
7
Infrastructure — Live views of Azure Blob Storage, Service Bus, Cosmos DB and App Insights. Assertions can verify blobs, queue messages, DB records and logs.
8
FP Teams — Integrated messaging. Colleagues can share PR links that open directly in the browser. Check the taskbar for notifications.
Connected Azure services
🔐 Key Vault — Secrets flow into request headers. Add a secret in Key Vault and it appears in the header value dropdown immediately.
⚙️ App Config — Host URLs, API versions, channel IDs. Add a key-value and it syncs to the request builder dropdowns. Filter by environment (dev/stg/prd).
📐 SwaggerHub — Model schemas power the Body editor, Model Mapper, and Flow node I/O pills.

Collections
Collections are groups of API requests organised by service. They live in the left sidebar.
Using a collection request
1
Click any request in the sidebar. The URL, method and path load into the Request Builder.
2
Switch to Headers — the Key column is a searchable dropdown of standard headers. The Value column shows 🔐 Key Vault secrets and ⚙️ App Config values combined.
3
Switch to Body — pick a SwaggerHub model from the dropdown. All fields render as editable inputs.
Flow Editor
Build visual flow diagrams of your service calls. Nodes are services, arrows are calls.
Building a flow
1
Open Flow under Tools. Select an existing flow or click + New.
2
Drag a service from the left palette onto the canvas. Use the endpoint dropdown on the node to select which operation.
3
Enable 🔗 Connect mode. Click an output port on one node, then an input port on another to draw an arrow.
4
Click the + Add mapping pill on any arrow to open the field mapper for that connection. PRO TIP
5
Hit 💾 Save to persist the flow state.
Model Mapper
Visually map fields between two swagger models. Used both as a standalone tool and embedded in Flow arrows.
Mapping fields
1
Select a Source and Target model from the dropdowns.
2
Click a field in the Source column to select it (highlights green).
3
Click the matching field in the Target column. A coloured bezier line connects them.
4
Add a description in the card that appears in the Description column on the right.
5
Click 💾 Save — in the Flow Editor this persists the mapping to the arrow.
6
Click ⬇ PNG to export the current mapping as a high-res image for documentation.
